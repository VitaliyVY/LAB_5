# Текст для захисту лабораторної

## 1) Як почати розповідь
Сьогодні я показую лабораторну роботу з Kafka Streams на Java.  
Ідея така: я беру поїздки з вхідного топіка `Topic1`, агрегую їх за датою поїздки і відправляю результати в 4 окремі Kafka-топіки.

## 2) Що саме реалізовано
Я реалізував 4 обчислення по днях:
1. Середня тривалість поїздки за день.
2. Кількість поїздок за день.
3. Найпопулярніша початкова станція за день.
4. Топ-3 станцій за день, де враховується і станція старту, і станція фінішу.

Кожне обчислення пишеться в окремий output topic:
- `trip-avg-duration-by-day`
- `trip-count-by-day`
- `trip-top-start-station-by-day`
- `trip-top3-stations-by-day`

## 3) Що показати у коді (по кроках)

### Крок 1. Налаштування застосунку
Відкриваю `TripStreamsApplication.java` і показую:
- читання `KAFKA_BOOTSTRAP_SERVERS`, `INPUT_TOPIC`, назв output topic
- запуск Kafka Streams

Що сказати:
“Тут задається підключення до Kafka і назви топіків, куди будуть записуватися результати”.

### Крок 2. Читання вхідного потоку
Показати частину, де:
- читається `Topic1`
- кожне повідомлення парситься в `TripRecord`

Що сказати:
“На цьому етапі я дістаю з повідомлення дату, тривалість, станцію старту і фінішу. Некоректні повідомлення пропускаються”.

### Крок 3. Обчислення кількості поїздок за день
Показати блок:
- `groupBy(day)` -> `count()` -> запис у `trip-count-by-day`

Що сказати:
“Для кожної дати додається +1 за кожну поїздку”.

### Крок 4. Обчислення середньої тривалості
Показати блок:
- `sum(duration)` + `count(day)` + `sum/count`
- запис у `trip-avg-duration-by-day`

Що сказати:
“Я окремо рахую суму тривалостей і кількість, потім обчислюю середнє”.

### Крок 5. Найпопулярніша стартова станція
Показати блок:
- підрахунок `day + from_station`
- вибір максимуму
- запис у `trip-top-start-station-by-day`

Що сказати:
“Для кожного дня визначаю станцію, яка найчастіше була стартовою”.

### Крок 6. Топ-3 станцій (старт + фініш)
Показати блок:
- для кожної поїздки враховується 2 станції: `from` і `to`
- підрахунок по днях
- сортування і `limit(3)`
- запис у `trip-top3-stations-by-day`

Що сказати:
“Тут я враховую обидві точки поїздки і формую трійку найпопулярніших станцій за день”.

### Крок 7. Docker і топіки
Показати `docker-compose.yml`:
- `kafka-init` створює всі потрібні топіки
- сервіс `streams-processor` запускає Java застосунок

Що сказати:
“Інфраструктура піднімається через Docker Compose, топіки створюються автоматично”.

## 4) Що показати під час демонстрації

### Команда запуску
```bash
docker compose up -d --build
```

### Показати, що сервіс працює
```bash
docker compose logs --tail=120 streams-processor
```

### Показати результати в кожному output topic
```bash
docker compose exec broker1 kafka-console-consumer --bootstrap-server broker1:9092 --topic trip-avg-duration-by-day --from-beginning --max-messages 10
docker compose exec broker1 kafka-console-consumer --bootstrap-server broker1:9092 --topic trip-count-by-day --from-beginning --max-messages 10
docker compose exec broker1 kafka-console-consumer --bootstrap-server broker1:9092 --topic trip-top-start-station-by-day --from-beginning --max-messages 10
docker compose exec broker1 kafka-console-consumer --bootstrap-server broker1:9092 --topic trip-top3-stations-by-day --from-beginning --max-messages 10
```

Що сказати:
“Я показую, що всі 4 обчислення реально формуються і потрапляють у свої топіки”.

## 5) Як пояснити повтори у повідомленнях

- реалізував 4 потрібні метрики,
- і відправив кожну метрику в окремий Kafka-топік.
