# Проста блок-схема (українською)

```mermaid
flowchart TD
    A([Початок]) --> B[Прочитати налаштування]
    B --> C[Читати повідомлення з Topic1]
    C --> D{Дані коректні?}

    D -- Ні --> D1[Пропустити]
    D1 --> C

    D -- Так --> E[Взяти з повідомлення:<br/>дату, тривалість,<br/>станцію старту, станцію фінішу]

    E --> F1[Порахувати кількість поїздок за день]
    E --> F2[Порахувати середню тривалість за день]
    E --> F3[Знайти найпопулярнішу стартову станцію за день]
    E --> F4[Знайти топ-3 станцій за день<br/>(старт + фініш)]

    F1 --> G1[Записати в topic:<br/>trip-count-by-day]
    F2 --> G2[Записати в topic:<br/>trip-avg-duration-by-day]
    F3 --> G3[Записати в topic:<br/>trip-top-start-station-by-day]
    F4 --> G4[Записати в topic:<br/>trip-top3-stations-by-day]

    G1 --> H[Чекати наступне повідомлення]
    G2 --> H
    G3 --> H
    G4 --> H

    H --> C
```

## Дуже коротко
- Програма постійно читає нові поїздки з `Topic1`.
- Для кожної дати оновлює статистику.
- Результати пише в 4 окремі топіки.
